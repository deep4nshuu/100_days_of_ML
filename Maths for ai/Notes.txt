## Overfitting : Means model is overtrained on training data. It happens when model learns detail & noise in training dataset to extent that it negatively impacts performance of model
- Optimal : Curve will be regular or generalised
- Overfit : Curve try to join/fit all data pts

# Sign of Overfitting : High Traning data Accuracy and Very Low Test data Accuracy

# Cause of Overfitting : 
Less Data, Increased Complexity of model when simple data, More no of layers in Neural Network

# Preventing Overfitting by :
Using more data, Reduce no of layers in Neural Network, Early stopping(stop training model when started overfitting), Bias-Variance Tradeoff(model optimization technique), Use Dropouts(used in NN to drop neuron for stopping overfitting)


## UnderFitting : It happens when model does not learn enough from data. So it occur when ml model cannot capture underlying trend of data
- Optimal : it tries to find trend and form curve a/c to that
- Underfit : it is not able to find trend so form a straight line(try to generalise all dataset which lead to high bias)

# Sign of Underfitting : Very Low Traning data Accuracy 

# Causes for Underfitting :
Choosing wrong model, less complexity of model, less variance but high bias

# Prevent Underfitting by :
Choosing correct model, Increasing complexity of model, more no of parameters to model, Bias-Variance Tradeoff


## Bias-Variance Tradeoff :
# Bias : the difference b/w avg predictns of our model and correct value which we are trying to predict 
- So when we use linear model there will be huge bias but no variance

# Variance : It is the amt that the estimate of target functn will change if different training data is used
- But when we use complex model, it will have 0 bias but high variance

- In Underfitting, High Bias & Low Variance
- In Overfitting, Low Bias & High Variance

# Take on : 
- When we use more complex model, it will cause overfitting as variance will be high but bias will be negligible
- But when we use simple model, model will be Underfit as it will have low variance but high bias.
- So we need to have middle complexity model having significant bias and variance val.

# Techniques to have better Bias-Variance Tradeoff :
Good model selectn, Regularizatn(reduce coefficient -> reduce complexity), Dimensionality Reductn(reducing features), Ensemble methods(multiple model to reduce Underfitting)


## Loss Functn : It measures how far an estimated value is from its true value.
- It is helpful in determining which model perform well & which parameters are better
- Lower the loss val  --> Higher effecient the model will be

# Formula : loss = (1/n)sigma(correct - predicted)^2


## ML Evaluation :
# Accuracy score : In classificatn, accuracy score is the ratio of no of correct predictns to the total no of input data points.
- Formula : A.S = (No of correct predictns / total no of data pts) * 100
- To use : from sklearn.metrics import accuracy_score

# Mean Squared Error : MSE measures the avg of the Squares of error, that is, the avg squared difference b/w the estimated value and actual val
- Formula :  MSE = (1/n)sigma(correct - predicted)^2
- To use : from sklearn.metrics import mean_squared_error


## Types of Parameters :
# Model Parameters : These are the parameters of model that can be determined by training with training data. These can be conisdered as Internal Parameters.
> Weights - Weight decides how much influence the Input will have on the Output (can be conisdered slope of line)
- So changing the Weight of line -> Will change the Orientation of line
> Bias - It is the offset value given to model. Bias is uesd to shift the model in a particular directn. It is similar to Y-intercept. 'b' is equal to 'Y' when all features value are zero.
- So changing Bias value -> mean changing the Y-intercept(val of y when x=0) of line

- Eq : Y = w*x + b   -> line can have multiple features
Here : x(feature/input), y(target/output), w(wt), b(bias)

# HyperParameters : These are parameters whose values control the learning process(fitting the data). These are adjustable parameters used to obtain an optimal model. These can be conisdered as External Parameters.

> Learning rate - the amt of change in the values of wt & bias to minimise the loss fn
- It is a turning parameter in an optimization algo that determines the step size at each iteratn while moving toward a minimum of a loss fn
- Value of learning rate is depend on mag of dataset and wt

> No of Epochs - No of iteration for training model again & again for finding better bias & wt value
- No of times the model iterates over the entire dataset.
- No of iteratn should be Optimal as low no -> Underfitting & Higher no -> Overfitting


## Model Optimization : 
Optimization refers to determining best parameters for a model, such that the loss fn of the model decreases, as a result of which the model can predict more accurately

# Gradient Descent
> G.D is an optimization algo used for minimizing the loss fn in various ml algorithms. It is used for updating the parameters of learning model.
> We started with random wt val & gradually Increasing/Decreasing wt based on initial pt and then the val of wt on which we have minimum loss fn that pt is global minimum and this process is called Gradient Descent
> Global Minimum : the pt where the value of loss fn is lowest

> Formula :  w(updated wt) = w(initial wt) - L * dw
and  b(updated bias) = b(initial bias) - L * db
Here, dw(partial derivative of loss fn wrt w) & db(partial derivative of loss fn wrt b)
